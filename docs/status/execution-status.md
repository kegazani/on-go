## Зафиксированные отложенные решения

1. Возврат к обучению более крупной pooled multi-dataset глобальной модели не является ближайшим шагом и сохранен как отдельный future track.
2. Этот future track можно открывать только после появления безопасных harmonized non-label signal features, общих для внешних datasets.
3. Второе обязательное условие: у внешних datasets должны появиться нормальные claim-grade labels; proxy-mapping недостаточен как основной supervision source.
4. До выполнения этих условий внешние datasets используются для harmonization, validation и transfer/cross-dataset проверок, а не как основание для большой общей модели.

## Правило перехода

Если пользователь пишет `перейдем к следующему шагу`, агент должен:

1. посмотреть этот файл;
2. найти шаг со статусом `next`;
3. назвать его пользователю;
4. кратко описать, что будет сделано;
5. дождаться подтверждения или корректировки.

## Как обновлять этот файл

После завершения шага:

1. текущий шаг переводится в `completed`;
2. следующий логический шаг переводится в `next`;
3. обновляется дата;
4. в `Результат` кратко фиксируется выход шага.
